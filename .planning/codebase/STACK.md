# Technology Stack

**Analysis Date:** 2026-03-04

## Project Type

**VCF (Vibe Coding Framework)** is a documentation-based system for building AI-powered tools. This is NOT an application — it is a structured framework delivered as markdown documents and templates.

---

## Languages

**Primary:**
- **Markdown (GitHub-flavored)** - All framework documentation, guides, checklists, forms, templates
- **YAML** - MkDocs configuration (`mkdocs.yml`)
- **JSON** - Configuration files (`mkdocs.yml` uses JSON for metadata)

**Secondary:**
- **JavaScript** - Search indexing via Lunr.js (auto-generated by MkDocs Material theme)
- **HTML/CSS** - Theme styling (auto-generated by MkDocs Material theme, assets in `site/assets/`)

---

## Runtime & Build System

**Documentation Generator:**
- **MkDocs** - Static site generator for markdown documentation
  - Theme: Material for MkDocs (`theme: material` in `mkdocs.yml`)
  - Search plugin enabled with Lunr
  - Markdown extensions: admonition, superfences, tabbed content, emoji, tables, TOC

**Build/Deployment:**
- Output: Static HTML site (pre-generated in `site/` directory)
- Hosting: Ready for any static hosting (GitHub Pages, Vercel, Netlify, S3, etc.)

## Configuration

**MkDocs Configuration (`mkdocs.yml`):**
```yaml
site_name: Vibe Coding Framework
site_description: A structured framework for building AI-powered tools...
theme:
  name: material
  palette: [light (indigo primary) | dark (slate scheme)]
  font: Inter (text), JetBrains Mono (code)
  features: [navigation.instant, navigation.tracking, search.suggest, ...]
markdown_extensions:
  - admonition (collapsible notes)
  - pymdownx.details (expandable <details> sections)
  - pymdownx.superfences (code blocks)
  - pymdownx.tabbed (tab switching)
  - tables, attr_list, toc
plugins:
  - search (full-text search)
```

**Claude Configuration (`.claude/settings.local.json`):**
- Permissions: WebSearch allowed
- Purpose: Enable Claude to research and validate content during framework development

**Build Output:**
- Generated site: `/site/` (pre-built static HTML)
- Not committed to git (standard practice for generated output)

## Documentation-as-Code Practices

**Version Control:**
- All markdown in `/docs/` tracked in git
- Single source of truth: markdown documents
- Build output (`/site/`) regenerated on deploy

**Modular Structure:**
- **Core framework (folders 01-06):** Universal, centrally maintained
- **Client config (folder 07):** Customized per client engagement
- **Tool configs (folder 08):** Optional accelerators for specific AI tools

**Progressive Disclosure Pattern:**
Every checklist item and guide step uses collapsible `<details>` sections:
```markdown
- [ ] **Step title**

    <details><summary>WHY</summary>
    One-sentence consequence for decision-makers
    </details>

    <details><summary>HOW TO CHECK</summary>
    Step-by-step instructions for implementers
    </details>
```

This allows three audience profiles (Builder/Explorer/Cautious) to access content at their level without choosing a label.

## Deployment Requirements

**Development:**
- MkDocs installed (Python package: `pip install mkdocs-material`)
- Git for version control
- Text editor or IDE with markdown support

**Production:**
- Static web server (any HTTP server works: Nginx, Apache, Node.js, etc.)
- HTTPS capable (recommended for all client instances)
- No backend/database needed (pure static files)
- Typical deployment: GitHub Pages, Vercel, Netlify, or custom domain

**CI/CD (if implemented):**
- Build hook: `mkdocs build` → generates `/site/`
- Deploy hook: Copy `/site/` contents to web server
- Triggers: On git push to main/production branch

---

## Platform Requirements

**Development:**
- Python 3.7+ (for MkDocs)
- Git 2.0+ (for version control)
- Any OS (macOS, Linux, Windows) with Python support
- No database, no server runtime needed

**For Builders Using VCF:**
The framework guides builders in selecting tools based on project needs:
- **Green Path:** Any tier (Lovable, Bolt, v0, Replit Agent, Cursor, Claude Code)
- **Yellow Path:** No-Code or Low-Code tier (Lovable, Bolt, v0, Replit Agent)
- **Orange/Red Paths:** Low-Code or Pro Tools (Replit Agent, Cursor, Claude Code)

## Not Applicable to This Project

- **Database:** None (documentation system, no data persistence)
- **APIs:** None (static documentation, no server endpoints)
- **Authentication:** None (documentation is public or company-internal)
- **Package Manager:** MkDocs via pip is the only dependency manager
- **Testing Framework:** None (documentation quality verified manually via checklists)
- **Monitoring:** None (static site generation, no runtime to monitor)
