1. Create `.github/workflows/ci.yml` with the official MkDocs Material workflow pattern, modified to use `requirements.txt`:

```yaml
name: ci
on:
  push:
    branches:
      - main
permissions:
  contents: write
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Configure Git Credentials
        run: |
          git config user.name github-actions[bot]
          git config user.email 41898282+github-actions[bot]@users.noreply.github.com
      - uses: actions/setup-python@v5
        with:
          python-version: 3.x
      - run: echo "cache_id=$(date --utc '+%V')" >> $GITHUB_ENV
      - uses: actions/cache@v4
        with:
          key: mkdocs-material-${{ env.cache_id }}
          path: ~/.cache
          restore-keys: |
            mkdocs-material-
      - run: pip install -r requirements.txt
      - run: mkdocs gh-deploy --force
```

Key points:
- Trigger: push to `main` only (matches Phase 1 branch rename)
- Permissions: `contents: write` (required for mkdocs gh-deploy to push to gh-pages)
- Uses `pip install -r requirements.txt` (NOT bare `pip install mkdocs-material`) per Phase 1 pinning decision
- Git config step required for `mkdocs gh-deploy` to commit on the runner
- `actions/cache@v4` with weekly cache key for pip dependencies
- `mkdocs gh-deploy --force` handles build + commit + push to gh-pages atomically

2. Update `mkdocs.yml` line `site_url: ""` to `site_url: https://vedantggwp.github.io/SP-VibeFrame/` so internal links (canonical URLs, navigation, sitemap) resolve correctly on the live site.

3. Commit both files and push to main. This triggers the first CI run.

4. Wait for the CI run to complete (poll with `gh run list --workflow ci.yml --limit 1`). If it fails, diagnose from logs (`gh run view --log-failed`) and fix.

5. After the CI run succeeds and the `gh-pages` branch exists (verify with `git ls-remote origin gh-pages`), enable GitHub Pages to serve from that branch:

```bash
gh api -X PUT "repos/vedantggwp/SP-VibeFrame/pages" \
  -f source='{"branch":"gh-pages","path":"/"}' \
  --header "Accept: application/vnd.github+json"
```

If the API call fails because Pages is already auto-enabled, verify with:
```bash
gh api repos/vedantggwp/SP-VibeFrame/pages --jq '{status: .status, url: .html_url}'
```

6. Wait up to 2-3 minutes for Pages to propagate, then verify the site is live:
```bash
curl -s -o /dev/null -w "%{http_code}" https://vedantggwp.github.io/SP-VibeFrame/
```
