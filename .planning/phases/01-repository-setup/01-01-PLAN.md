<task type="auto">
  <name>Task 2: Rename branch to main, create private GitHub repo, and push</name>
  <files></files>
  <action>
1. Rename the local branch from master to main:
   ```
   git branch -m master main
   ```
   This aligns with GitHub's default branch naming and prevents friction in Phase 2 when the CI workflow targets `main`.

2. Check if a remote repo already exists to avoid a naming conflict:
   ```
   gh repo list vedantggwp --json name --jq '.[].name' | grep -x 'SP-VibeFrame'
   ```
   If it already exists, skip creation and just add the remote + push. If it does NOT exist, proceed to step 3.

3. Create the private GitHub repository and push in one command:
   ```
   gh repo create SP-VibeFrame \
     --private \
     --source=. \
     --remote=origin \
     --push \
     --description "Vibe Coding Framework - private documentation site"
   ```
   The `--source=.` flag creates the remote from the local directory and `--push` pushes all commits. This avoids the "diverged history" issue that occurs when GitHub creates an initial README commit.

4. Verify the repo is private:
   ```
   gh repo view vedantggwp/SP-VibeFrame --json visibility --jq '.visibility'
   ```
   Must output `PRIVATE`.

5. Verify all local commits are pushed (nothing pending):
   ```
   git log origin/main..HEAD --oneline
   ```
   Must output nothing (empty = fully synced).

6. Verify the remote is correctly configured:
   ```
   git remote -v
   ```
   Must show origin pointing to github.com/vedantggwp/SP-VibeFrame.
  </action>
  <verify>
    <automated>gh repo view vedantggwp/SP-VibeFrame --json visibility --jq '.visibility' | grep -q 'PRIVATE' && test -z "$(git log origin/main..HEAD --oneline)" && echo "PASS" || echo "FAIL"</automated>
  </verify>
  <done>Private GitHub repo vedantggwp/SP-VibeFrame exists. Local branch is main. All commits are pushed to origin/main. Remote visibility is PRIVATE.</done>
</task>
