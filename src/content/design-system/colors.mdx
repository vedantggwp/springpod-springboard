## Status colors

System feedback colors for confirmations, warnings, informational messages, and errors. Each has a light variant (for backgrounds) and a dark variant (for text and icons).

| Status | Light | Dark | Usage |
|--------|-------|------|-------|
| Success | `#BFF3A5` | `#357E10` | Confirmations, completed states |
| Warning | `#FEE3AF` | `#8C5E03` | Cautions, pending actions |
| Info | `#C2E1FF` | `#004A8F` | Informational messages |
| Error | `#f8b5bb` | `#820D19` | Errors, validation failures |

<Collapsible variant="example" title="Example: status message styling">

A success message would use `#BFF3A5` as the background and `#357E10` as the text colour. This gives enough contrast for accessibility while clearly signaling "all good" to the user.

An error message would use `#f8b5bb` as the background and `#820D19` as the text â€” immediately recognisable as something that needs attention.

</Collapsible>

## Dark mode

Colors adapt automatically via CSS custom properties. The primary brand colors (teal, navy, blue) stay consistent across light and dark modes. Layout tokens â€” backgrounds, borders, and shadows â€” switch to their dark variants when the user toggles dark mode.

You do not need to define separate dark-mode colors when building with SpringBoard's design tokens. The system handles it.

## Usage guidelines

<Admonition type="tip" title="How to use these colors">

- **Always use token classes** (`text-sp-navy`, `bg-sp-teal`) instead of raw hex values in code.
- **Reference this page** when building tools to match Springpod's visual identity.
- **When using AI tools** (Lovable, v0, Replit), paste the hex values from this page into your prompts so the generated UI matches our brand.
- **Accessibility:** Ensure sufficient contrast between text and background colors. Use dark variants on light backgrounds and light variants on dark backgrounds.

</Admonition>
