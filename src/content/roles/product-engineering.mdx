## Common Scenarios

<Collapsible variant="tip" title="I'm adding AI to a student-facing feature">
Red path. This requires a safeguarding review, AI testing (prompt injection, output safety), and a pen test. Start with the [Intake Quiz](/intake/project-intake) and expect a full review cycle. Budget extra time — this is the highest-scrutiny path for good reason.
</Collapsible>

<Collapsible variant="tip" title="I need to integrate a new third-party API">
Orange path minimum. Check [Security Standards](/standards/security) for API requirements (auth, rate limiting, data in transit) and file a [Review Request](/forms/review-request) before going live. If the API handles student PII, this may escalate to Red.
</Collapsible>

<Collapsible variant="tip" title="I'm building an internal tool that queries the database">
Green or Yellow depending on data sensitivity. If you are querying anonymised or aggregate data, Green is fine. If the tool accesses student PII or partner data, check [Data Workflows](/standards/data-workflows) for classification — you may need Yellow or Orange controls.
</Collapsible>
